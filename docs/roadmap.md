# 版本路线图

## v0.1.0 - 项目定义

目标：把产品方向、学习闭环和迭代方式固定下来。

- 建立 Git 仓库。
- 创建 README、产品规格和版本路线图。
- 创建 GitHub private repo 设置指南。
- 明确 MVP 范围、内容版权边界和技术方向。
- 后续开始原型设计前，所有新增想法先记录到文档或 issue。

验收标准：

- 项目可以通过 Git 追踪。
- 项目知道如何安全连接 GitHub private repo。
- 下一阶段开发者能理解产品目标和 MVP 功能。

## v0.2.0 - iOS 静态原型

目标：做出可点击但不接真实 AI 的 SwiftUI 原型。

- 状态：已启动，SwiftUI 静态原型骨架已创建，并已在 iPhone 模拟器运行成功。
- 今日学习页。
- 主题选择页。
- 听力跟读页。
- 句子听写/默写原型。
- 今日页展示完成状态、完成进度、5 句精听、关键单词和日常语法结构。
- 跟读页按跟读、听写、默写的顺序逐步展示。
- 素材详情页展示不同类型素材的 mock 视频/歌词/新闻/文章内容。
- 复盘页展示基于“主线进阶 + 弱项复习 + 兴趣调味”的明日推荐 mock 结果。
- 口语反馈页。
- 学习复盘页。
- 素材主题展示从日常表达到新闻/报刊读物的进阶路径。

验收标准：

- 可以在 iPhone 模拟器跑通完整 15 分钟学习流程的页面跳转。
- 所有页面使用 mock 数据。

## v0.3.0 - AI 听说闭环

目标：接入最小可用的后端和 AI 能力。

- 后端代理 AI 调用，客户端不保存 API key。
- 支持录音上传和语音转写。
- 支持基于转写文本生成口语反馈。
- 支持听写/默写答案校验、漏词标记和自然表达建议。
- 支持保存一次完整学习记录。
- 支持根据学习表现推荐下一阶段内容难度。
- 支持每日推荐引擎：根据昨日弱项、当前阶段和兴趣素材生成明日 15 分钟任务。

验收标准：

- 用户完成一次跟读后能看到转写、评分和改进建议。
- 网络失败时有明确提示，不丢失本地录音状态。

## v0.4.0 - 兴趣内容系统

目标：让学习内容更贴近剧集和歌曲兴趣。

- 支持搜索剧集或歌曲主题。
- 支持保存素材元数据和公开视频链接。
- 支持手动粘贴合法短片段生成学习任务。
- 支持 AI 生成相似生活情景对话。

验收标准：

- 用户可以围绕一个喜欢的作品主题生成当天学习任务。
- App 不保存完整剧本、完整歌词或盗版字幕。

## v0.5.0 - 个人 Beta

目标：让 App 具备连续使用价值。

- 学习日历。
- 连续学习天数。
- 口语评分趋势。
- 词块复习。
- 历史记录搜索。

验收标准：

- 连续使用 7 天后能看到清晰的进步记录。
- 能回看过去学习过的表达和反馈。

## v1.0.0 - 稳定自用版

目标：达到稳定、顺手、可长期个人使用。

- 修复核心崩溃和卡顿。
- 完善隐私说明。
- 完善数据备份策略。
- 整理配置文档。
- 固化发布流程和 tag 规范。

验收标准：

- 能稳定运行一周以上。
- 每日学习流程少于 3 次点击即可开始。
- API key、录音和个人学习数据不会被误提交到 Git。
